▸ ЧТО ТАКОЕ MC-PLUGIN?
  MC-Plugin — это многофункциональный плагин для Minecraft (Paper 1.21.4+),
  который добавляет на сервер системы энергии, реакторов, крафта, наказаний,
  вайтлиста, блэклиста, кастомного чата, таба, скорборда, MOTD, ваниша,
  авторизации, радиации и многое другое.

  Все команды управления — через /mp <подкоманда>.
  Полный список команд: /mp help

▸ ФАЙЛЫ КОНФИГУРАЦИИ

  1. plugins/MC-Plugin/config.yml      — основные настройки плагина
  2. plugins/MC-Plugin/messages.yml    — все сообщения (для перевода)
  3. plugins/MC-Plugin/plugin-guide.txt — этот файл-инструкция

  config.yml и messages.yml содержат ТОЛЬКО значения — без комментариев.
  Вся документация по настройке — в этом файле (plugin-guide.txt).

╔══════════════════════════════════════════════════════╗
║           СИСТЕМА КОНФИГУРАЦИИ                       ║
╚══════════════════════════════════════════════════════╝

  Если в вашем config.yml не хватает каких-то ключей
  (например после обновления плагина) — плагин АВТОМАТИЧЕСКИ
  добавит недостающие значения в КОНЕЦ файла.

  Все ваши настройки СОХРАНЯТСЯ — ничего не сбросится.
  Недостающие ключи добавляются с комментарием:
    # === Missing keys added by MC-Plugin (auto-repair) ===

  Если этот файл (plugin-guide.txt) устарел — плагин
  перезапишет его своей актуальной версией при следующем запуске.

────────────────────────────────────────────────
 1. ⚡ СИСТЕМА ЭНЕРГИИ (energy)
────────────────────────────────────────────────

  energy.generator:
    enabled              — вкл/выкл генератор энергии
    energy_per_fuel      — сколько энергии даёт 1 топливо за тик
    fuel_burn_ticks      — сколько тиков горит 1 топливо (100 = 5 сек)
    log                  — логировать работу

  energy.electric_furnace:
    enabled              — вкл/выкл электрическую печку
    energy_per_smelt     — энергия за 1 переплавку
    cooldown_ms          — кулдаун между молниями (мс)

  energy.cable:
    enabled              — вкл/выкл кабели
    max_energy           — макс энергия кабеля (0 = не хранит)
    log                  — логировать
    blink.enabled        — мигание кабеля при передаче
    blink.off_ticks      — тиков в выкл состоянии
    blink.on_ticks       — тиков во вкл состоянии

  energy.battery:
    max_energy           — макс энергия батареи (100000)
    discharge_per_tick   — отдача в сеть за тик при редстоун-сигнале
    smooth_charge.enabled     — плавная динамическая зарядка
    smooth_charge.charge_multiplier    — множитель скорости зарядки
    smooth_charge.discharge_multiplier — множитель скорости разрядки
    visual.particles_enabled  — частицы электричества

  energy.battery_drain:
    enabled              — вкл/выкл (устарело)
    log                  — логировать

  energy.balancer:
    enabled              — вкл/выкл балансировщик энергии
    include_batteries    — включать батареи в балансировку
    max_transfer         — макс перенос за тик
    log                  — логировать

 2. 💻 СИСТЕМА КОДОВОЙ ПАНЕЛИ (codepanel)
────────────────────────────────────────────────

  codepanel:
    max_length           — макс длина пароля (10)
    empty_char           — символ пустого места ("-")
    keyboard_offset      — сдвиг клавиатуры (2)

    colors               — цвета элементов панели
    buttons              — цвета кнопок
    sounds               — звуки панели

  Все ключи кодовой панели хранятся в SQLite (database.db),
  управляются через /mp codepane key add|remove|modify|list

  /mp help                — справка

  ───── ЭНЕРГИЯ ─────
  /mp power off           — выключить сервер
  /mp power reboot        — перезагрузить сервер
  /mp power undo          — отменить запрос

  ───── ТЕЛЕПОРТАЦИЯ ─────
  /mp chgdim [мир]        — сменить измерение
  /mp askcords <ник>      — запросить координаты

  ───── ДОМА ─────
  /mp sethome <название>  — установить дом
  /mp home <название>     — телепортироваться домой
  /mp delhome <название>  — удалить дом
  /mp listhomes           — список домов
  /mp ophomels <ник>      — список домов игрока (op)
  /mp opdelhome <ник> <название> — удалить дом игрока (op)

  ───── НАКАЗАНИЯ ─────
  /mp punish ban <ник> <причина> [-time:N] [-permanent] [-ip] [-hw]
  /mp punish mute <ник> <причина> [-time:N] [-permanent] [-ip] [-hw]
  /mp punish kick <ник> <причина> [-ip] [-hw]
  /mp punish warn <ник> <причина> [-time:N] [-permanent] [-ip] [-hw]
  /mp punish listwarns [ник]
  /mp punish unban <ник>
  /mp punish unmute <ник>
  /mp punish unwarn <ник> <причина> <warnId>

  Флаги:
    -time:<N>s|m|h|d  — временное наказание
    -permanent         — перманентное
    -ip                — наказание по IP
    -hw                — наказание по железу

  ───── ВАЙТЛИСТ / БЛЭКЛИСТ ─────
  /mp whitelist on|off|add <ник>|remove <ник>|list
  /mp blacklist on|off|add <ник>|remove <ник>|list

  ───── OP ВАЙТЛИСТ ─────
  /mp opwhitelist on|off|add <ник>|remove <ник>|list

  ───── ТЕХРАБОТЫ ─────
  /mp maint on|off|whitelist add <ник>|whitelist remove <ник>|whitelist list

  ───── ПРОЧЕЕ ─────
  /mp vanish <ник>       — включить/выключить ваниш
  /mp suicide            — суицид (с подтверждением)
  /mp togglebb           — вкл/выкл боссбар
  /mp togglesb           — вкл/выкл скорборд
  /mp togglefly          — вкл/выкл полёт
  /mp checkrad [ник]     — проверить радиацию
  /mp setrad <ник> <N>   — установить радиацию
  /mp toggleradview      — вкл/выкл дозиметр
  /mp codepane [название] — открыть кодовую панель
  /mp notes              — открыть заметки
  /mp checkver           — проверить обновления
  /mp bc <сообщение>     —广播 сообщение
  /mp item int set|add|list <ник> <предмет> [кол-во]

  ───── ВАНИЛЬНЫЙ /whitelist ─────
  Ванильная команда /whitelist ЗАБЛОКИРОВАНА.
  Используйте: /mp whitelist on|off|add|remove|list

  ───── ВАНИЛЬНЫЙ /stop и /restart ─────
  Перехвачены плагином.
  Используйте: /mp power off и /mp power reboot

  ───── БАЗА ДАННЫХ ─────
  Все данные хранятся в plugins/MC-Plugin/database.db (SQLite).
  Регулярное резервное копирование рекомендуется.

  ───── ДАТАПАК ─────
  Плагин автоматически устанавливает датапак MC-Datapack
  в папку world/datapacks/ при первом запуске.
