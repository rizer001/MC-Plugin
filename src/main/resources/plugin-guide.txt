▸ ЧТО ТАКОЕ MC-PLUGIN?
  MC-Plugin — многофункциональный плагин для Paper 1.21.4+.
  Добавляет: энергию, реакторы, крафты, наказания, вайтлист, блэклист,
  кастомный чат, таб, скорборд, MOTD, ваниш, авторизацию, радиацию,
  беспроводной редстоун, защиту от ботов, фильтр чата и многое другое.

  Все команды: /mp <подкоманда>
  Справка:      /mp help

▸ ФАЙЛЫ КОНФИГУРАЦИИ

  plugins/MC-Plugin/config.yml       — ОСНОВНЫЕ НАСТРОЙКИ (все значения)
  config.yml (секции messages: + messages_en:) — СООБЩЕНИЯ
  config.yml (раздел plugin-guide в НАЧАЛЕ файла) — документация

  config.yml содержит ВСЁ: значения, локализованные сообщения, ДОКУМЕНТАЦИЮ (в начале).
  ВСЯ документация по настройке — в НАЧАЛЕ этого файла (config.yml)

▸ АВТО-РЕМОНТ КОНФИГА
  Если в config.yml не хватает каких-то ключей (после обновления плагина) —
  плагин АВТОМАТИЧЕСКИ добавит недостающие значения в КОНЕЦ файла.
  Ваши настройки НЕ СБРОСЯТСЯ.

  Также если встроенный гайд устарел — плагин перезапишет
  его актуальной версией при следующем запуске.

▸ КАК ЧИТАТЬ ГАЙД
  Для каждой секции config.yml указано:
    • КЛЮЧ — путь в YAML (напр energy.generator.enabled)
    • Описание — что делает
    • Допустимые значения — что можно писать
    • Значение по умолчанию — что будет, если ключа нет
    • Примечания — важные нюансы

║   1. ⚡ СИСТЕМА ЭНЕРГИИ (energy)                                 ║
║                                                                  ║
╚══════════════════════════════════════════════════════════════════╝

  Система энергии — основа плагина. Генераторы производят энергию,
  кабели передают, батареи хранят, печи и верстаки потребляют.

  ────────────────────────────────────────────────────────────────
  1.1. energy.generator — ГЕНЕРАТОР ЭНЕРГИИ
  ────────────────────────────────────────────────────────────────

  energy.generator.enabled
    • Описание: Включает/выключает генератор энергии.
      Генератор — блок, который сжигает топливо (уголь, древесина, лава
      и т.д.) и производит энергию.
    • Значения: true / false
    • По умолчанию: true

  energy.generator.energy_per_fuel
    • Описание: Сколько энергии (EU/тик) даёт одна единица топлива.
      Например: 100 — уголь даёт 100 EU за каждый тик горения.
      Чем выше число, тем быстрее копится энергия.
    • Значения: любое целое число ≥ 1
    • По умолчанию: 100

  energy.generator.fuel_burn_ticks
    • Описание: Сколько тиков (1 тик = 0.05 сек) горит 1 единица топлива.
      100 тиков = 5 секунд.
      Влияет на то, как часто нужно подбрасывать топливо.
    • Значения: любое целое число ≥ 1
    • По умолчанию: 100

  energy.generator.log
    • Описание: Логировать работу генератора в консоль.
      Полезно для отладки, но засоряет консоль на больших серверах.
    • Значения: true / false
    • По умолчанию: false

  ────────────────────────────────────────────────────────────────
  1.2. energy.electric_furnace — ЭЛЕКТРИЧЕСКАЯ ПЕЧЬ
  ────────────────────────────────────────────────────────────────

  energy.electric_furnace.enabled
    • Описание: Включает электрическую печь — плавит руду за энергию,
      а не за топливо. Работает быстрее обычной печи.
    • Значения: true / false
    • По умолчанию: true

  energy.electric_furnace.energy_per_smelt
    • Описание: Сколько энергии тратится на ОДНУ переплавку предмета.
      Пример: 100 — переплавка одной железной руды стоит 100 EU.
    • Значения: любое целое число ≥ 1
    • По умолчанию: 100

  energy.electric_furnace.cooldown_ms
    • Описание: Кулдаун между переплавками в миллисекундах.
      1000 мс = 1 секунда.
      Защищает от слишком быстрой переплавки (может лагать).
    • Значения: любое целое число ≥ 0
    • По умолчанию: 1000

  ────────────────────────────────────────────────────────────────
  1.3. energy.cable — КАБЕЛИ ПЕРЕДАЧИ
  ────────────────────────────────────────────────────────────────

  energy.cable.enabled
    • Описание: Включает кабели — соединяют генераторы, батареи и
      потребители в единую сеть. Без кабелей энергия не передаётся.
    • Значения: true / false
    • По умолчанию: true

  energy.cable.max_energy
    • Описание: Максимальная энергия, которую может хранить ОДИН кабель.
      0 — кабель НЕ хранит энергию (только передаёт).
      Если поставить > 0, кабель работает как мини-батарея.
    • Значения: 0 или любое целое число ≥ 1
    • По умолчанию: 0

  energy.cable.log
    • Описание: Логировать передачу энергии.
    • Значения: true / false
    • По умолчанию: false

  energy.cable.blink.enabled
    • Описание: Мигание кабеля при передаче энергии (визуальный эффект).
      Кабель меняет блок-стейт при передаче — выглядит как пульсация.
    • Значения: true / false
    • По умолчанию: true

  energy.cable.blink.off_ticks
    • Описание: Сколько тиков кабель находится в "выключенном" состоянии
      при мигании. Меньше = быстрее мерцание.
    • Значения: любое целое число ≥ 1
    • По умолчанию: 10

  energy.cable.blink.on_ticks
    • Описание: Сколько тиков кабель находится во "включённом" состоянии
      при мигании.
    • Значения: любое целое число ≥ 1
    • По умолчанию: 10

  ────────────────────────────────────────────────────────────────
  1.4. energy.battery — БАТАРЕЯ
  ────────────────────────────────────────────────────────────────

  energy.battery.max_energy
    • Описание: Максимальный запас энергии в ОДНОЙ батарее.
      100000 = 100k EU. При превышении батарея перестаёт принимать энергию.
    • Значения: любое целое число ≥ 1
    • По умолчанию: 100000

  energy.battery.discharge_per_tick
    • Описание: Сколько энергии батарея отдаёт в сеть ЗА ОДИН ТИК
      при наличии редстоун-сигнала. 10 — медленная разрядка,
      1000 — мгновенная отдача. Если 0 — батарея не отдаёт энергию.
    • Значения: любое целое число ≥ 0
    • По умолчанию: 10

  energy.battery.smooth_charge.enabled
    • Описание: Плавная динамическая зарядка.
      Вместо мгновенного пополнения — энергия прибывает постепенно.
      Визуально приятнее и реалистичнее.
    • Значения: true / false
    • По умолчанию: true

  energy.battery.smooth_charge.charge_multiplier
    • Описание: Множитель скорости зарядки при плавном режиме.
      2.0 — в 2 раза быстрее стандартной скорости.
      0.5 — в 2 раза медленнее.
    • Значения: любое дробное число ≥ 0.1
    • По умолчанию: 2.0

  energy.battery.smooth_charge.discharge_multiplier
    • Описание: Множитель скорости разрядки при плавном режиме.
    • Значения: любое дробное число ≥ 0.1
    • По умолчанию: 0.5

  energy.battery.visual.particles_enabled
    • Описание: Частицы электричества вокруг батареи при зарядке/разрядке.
      Может вызвать лаги при большом количестве батарей.
    • Значения: true / false
    • По умолчанию: true

  ────────────────────────────────────────────────────────────────
  1.5. energy.battery_drain — УСТАРЕВШИЙ РАЗРЯД БАТАРЕИ
  ────────────────────────────────────────────────────────────────

  energy.battery_drain.enabled
    • Описание: Устаревшая механика. Рекомендуется использовать
      smooth_charge вместо этого. Оставлено для обратной совместимости.
    • Значения: true / false
    • По умолчанию: true

  energy.battery_drain.transfer_per_tick
    • Описание: Сколько энергии передаётся за тик.
    • Значения: любое целое число ≥ 1
    • По умолчанию: 5

  energy.battery_drain.log
    • Описание: Логировать разряд.
    • Значения: true / false
    • По умолчанию: false

  ────────────────────────────────────────────────────────────────
  1.6. energy.balancer — БАЛАНСИРОВЩИК ЭНЕРГИИ
  ────────────────────────────────────────────────────────────────

  energy.balancer.enabled
    • Описание: Балансировщик распределяет энергию равномерно между
      всеми потребителями в сети. Без него энергия идёт к первому
      подключённому потребителю, остальные могут голодать.
    • Значения: true / false
    • По умолчанию: true

  energy.balancer.include_batteries
    • Описание: Включать ли батареи в балансировку.
      Если false — батареи заряжаются в первую очередь, остаток идёт
      потребителям. Если true — энергия распределяется равномерно
      между батареями и потребителями.
    • Значения: true / false
    • По умолчанию: false

  energy.balancer.max_transfer
    • Описание: Максимальное количество энергии, которое балансировщик
      может перераспределить за один тик.
      25 — мягкое ограничение. 0 — без лимита (может вызвать лаги).
    • Значения: любое целое число ≥ 0
    • По умолчанию: 25

  energy.balancer.log
    • Описание: Логировать работу балансировщика.
    • Значения: true / false
    • По умолчанию: false

  ────────────────────────────────────────────────────────────────
  1.7. energy_crafting — ЭНЕРГИЯ ДЛЯ КРАФТА
  ────────────────────────────────────────────────────────────────

  energy_crafting.enabled
    • Описание: Некоторые кастомные крафты требуют энергию.
      Если выключено — крафты бесплатны (но могут не работать).
    • Значения: true / false
    • По умолчанию: true

  energy_crafting.energy_per_craft
    • Описание: Сколько энергии тратится на один крафт.
      100 — один кастомный крафт стоит 100 EU.
    • Значения: любое целое число ≥ 1
    • По умолчанию: 100

  energy_crafting.workbench_search_radius
    • Описание: Радиус поиска энергетической сети вокруг верстака
      (в блоках). 3 — ищет источники энергии в радиусе 3 блоков.
    • Значения: любое целое число ≥ 1
    • По умолчанию: 3

║   2. 💻 СИСТЕМА КОДОВОЙ ПАНЕЛИ (codepanel)                      ║
║                                                                  ║
╚══════════════════════════════════════════════════════════════════╝

  Кодовая панель — графический интерфейс для ввода кода/пароля.
  Используется в системе авторизации и других механиках.

  codepanel.max_length
    • Описание: Максимальная длина вводимого кода (в символах).
      10 — можно ввести до 10 символов. Больше 10 — может не влезть в GUI.
    • Значения: любое целое число от 1 до 20
    • По умолчанию: 10

  codepanel.empty_char
    • Описание: Символ, отображаемый на пустых позициях ввода.
      Обычно дефис "-" или подчёркивание "_".
    • Значения: любой ОДИН символ
    • По умолчанию: "-"

  codepanel.keyboard_offset
    • Описание: Сдвиг клавиатуры относительно центра экрана (в слотах).
      2 — стандартный отступ. Если панель выглядит съехавшей — меняйте.
    • Значения: любое целое число ≥ 0
    • По умолчанию: 2

  codepanel.loading.step_ticks
    • Описание: Длительность одного шага анимации загрузки (в тиках).
      5 тиков = 0.25 сек. Меньше = быстрее анимация.
    • Значения: любое целое число ≥ 1
    • По умолчанию: 5

  codepanel.loading.step
    • Описание: Количество шагов анимации загрузки перед открытием панели.
      10 шагов = около 2.5 секунд.
    • Значения: любое целое число ≥ 1
    • По умолчанию: 10

  codepanel.loading.delay_after
    • Описание: Задержка ПОСЛЕ анимации загрузки (в тиках) перед
      фактическим открытием панели. 20 тиков = 1 секунда.
    • Значения: любое целое число ≥ 0
    • По умолчанию: 20

  codepanel.enter_cooldown
    • Описание: Кулдаун между нажатиями кнопки Enter (в секундах).
      Защита от спама попытками ввода кода.
    • Значения: любое целое число ≥ 0
    • По умолчанию: 3

  codepanel.colors
    • Описание: Цвета элементов панели.
    • brackets:   цвет скобок вокруг поля ввода (DARK_RED)
    • code:       цвет вводимых символов (WHITE)
    • Значения: любой Bukkit Color (ChatColor) — RED, GREEN, DARK_RED, WHITE и т.д.

  codepanel.buttons
    • Описание: Цвета кнопок панели.
    • normal:     цвет обычных цифр/букв (WHITE)
    • reset:      цвет кнопки сброса (DARK_RED)
    • enter:      цвет кнопки подтверждения (DARK_GREEN)

  codepanel.sounds
    • Описание: Звуки для разных действий на панели.
    • digit:      звук нажатия цифры
    • backspace:  звук удаления символа
    • reset:      звук сброса всего кода
    • enter:      звук нажатия Enter
    • click:      звук клика по любой кнопке
    • step:       звук шага анимации загрузки
    • finish:     звук окончания анимации
    • success:    звук успешного ввода кода
    • fail:       звук неверного кода
    • Значения: названия звуков Minecraft (см https://hub.spigotmc.org/javadocs/bukkit/org/bukkit/Sound.html)
      Например: UI_BUTTON_CLICK, BLOCK_NOTE_BLOCK_PLING, ENTITY_PLAYER_LEVELUP

║   3. ⚛ РЕАКТОР ТЁМНОГО СИНТЕЗА (reactor)                       ║
║                                                                  ║
╚══════════════════════════════════════════════════════════════════╝

  Реактор — сложная механика с температурой, давлением, износом
  и возможностью взрыва (мелтдаун). Требует охлаждения и контроля.

  reactor.enabled
    • Описание: Включает реактор. Если выключено — блок реактора
      не работает (но не исчезает из мира).
    • Значения: true / false
    • По умолчанию: true

  reactor.temp_decay_rate
    • Описание: Скорость естественного спада температуры ядра (за тик).
      1 — медленно остывает. Больше — быстрее остывает.
    • Значения: любое целое число ≥ 0
    • По умолчанию: 1

  reactor.core_temp_max
    • Описание: Максимальная температура ядра (в градусах).
      6000 — при превышении начинается разрушение оболочки.
    • Значения: любое целое число
    • По умолчанию: 6000

  reactor.core_temp_min
    • Описание: Минимальная температура ядра.
      -272 — почти абсолютный ноль.
    • Значения: любое целое число
    • По умолчанию: -272

  reactor.core_temp_cool_min
    • Описание: Минимальная температура, до которой можно охладить ядро
      системами охлаждения.
    • Значения: любое целое число
    • По умолчанию: -270

  reactor.heat_rate
    • Описание: Скорость нагрева ядра при работе (за тик).
      3 — медленный нагрев. 10 — быстрый нагрев.
    • Значения: любое целое число ≥ 0
    • По умолчанию: 3

  reactor.cool_rate
    • Описание: Скорость охлаждения ядра системами охлаждения (за тик).
      3 — медленное охлаждение. Больше — эффективнее охлаждение.
    • Значения: любое целое число ≥ 0
    • По умолчанию: 3

  reactor.core_press_reduce_rate
    • Описание: Скорость снижения давления в ядре (за тик).
      1 — медленно. Больше — давление падает быстрее.
    • Значения: любое целое число ≥ 0
    • По умолчанию: 1

  reactor.case_temp_heat_rate
    • Описание: Скорость нагрева КОРПУСА реактора от ядра (за тик).
      2 — корпус греется медленнее ядра.
    • Значения: любое целое число ≥ 0
    • По умолчанию: 2

  reactor.case_temp_max
    • Описание: Максимальная температура корпуса.
      8000 — при превышении корпус начинает разрушаться.
    • Значения: любое целое число
    • По умолчанию: 8000

  reactor.case_temp_cool_rate
    • Описание: Скорость охлаждения корпуса (за тик).
    • Значения: любое целое число ≥ 0
    • По умолчанию: 2

  reactor.case_temp_cool_min
    • Описание: Минимальная температура корпуса.
    • Значения: любое целое число
    • По умолчанию: -271

  reactor.case_temp_decay_rate
    • Описание: Скорость спада температуры корпуса (за тик).
    • Значения: любое целое число ≥ 0
    • По умолчанию: 1

  reactor.case_press_heat_rate
    • Описание: Скорость роста давления в корпусе от температуры (за тик).
    • Значения: любое целое число ≥ 0
    • По умолчанию: 4

  reactor.case_press_max
    • Описание: Максимальное давление корпуса.
      10000 — при превышении корпус разрушается.
    • Значения: любое целое число ≥ 0
    • По умолчанию: 10000

  reactor.case_press_decay_rate
    • Описание: Скорость спада давления в корпусе (за тик).
    • Значения: любое целое число ≥ 0
    • По умолчанию: 1

  reactor.shell_integrity_decay_temp
    • Описание: Температура ядра, при КОТОРОЙ начинает разрушаться
      оболочка ядра. Выше этой температуры — оболочка теряет прочность.
    • Значения: любое целое число
    • По умолчанию: 5000

  reactor.shell_int_decay_rate
    • Описание: Скорость разрушения оболочки при превышении температуры
      (в единицах целостности за тик).
    • Значения: любое целое число ≥ 0
    • По умолчанию: 1

  reactor.shell_int_recovery_temp_max

    • Описание: Позволяет ломать неразрушимые блоки (бедрок, барьер)

    • По умолчанию: true

    • Описание: Интервал между "ударами" по блоку (в тиках).
      4 — 5 ударов в секунду.
    • Значения: любое целое число ≥ 1
    • По умолчанию: 4

  features.unbreakable_breaker.blocks.<МАТЕРИАЛ>:
    • Каждый тип блока (BEDROCK, BARRIER) настраивается отдельно.
    • enabled: можно ли ломать этот тип
    • max_damage: сколько "ударов" нужно для разрушения
    • min_tool_tier: мин уровень инструмента ("-" = любой)

    • break_naturally: ломается ли натурально (без частиц/звуков)
    • play_effects: играть ли эффекты/частицы
    • damage.<тип_кирки>: урон за удар конкретной киркой
    • efficiency_multiplier: множитель от Efficiency
    • haste_multiplier: множитель от Haste

  Пример для BEDROCK:
    damage:
      netherite_pickaxe: 4.0    — 4 ед урона за удар (25 ударов)
      diamond_pickaxe: 2.5
      default: 0.3              — 0.3 ед (334 удара кулаком)

    • Описание: Интервал проверки целостности (в тиках).

    • Описание: Множитель стоимости ремонта.
      1.0 — стандартная стоимость. 2.0 — в 2 раза дороже.
    • Значения: дробное число ≥ 0
    • По умолчанию: 1.0

  features.integrity.gradient
    • Описание: Цвета градиента целостности в описании предмета.
    • high_color: цвет высокой целостности ("#006600" — зелёный)
    • low_color: цвет низкой целостности ("#990000" — красный)

    • Описание: Текст перед числом целостности в описании предмета.

    • По умолчанию: "§fIntegrity:"

  features.integrity.on_break
    • Описание: Поведение при поломке предмета.
    • play_sound: играть звук поломки (true/false)
    • send_message: писать в чат "предмет сломался" (true/false)
    • sound: какой звук (ENTITY_ITEM_BREAK)
    • sound_volume: громкость (2.0)
    • sound_pitch: высота тона (1.0)

  features.integrity.logging
    • Описание: Логирование системы целостности.
    • log_init, log_break, log_errors — что логировать.

  features.integrity.anvil_repair
    • Описание: Ремонт в наковальне.
    • enabled: можно ли ремонтировать (true)
    • integrity_multiplier: сколько целостности восстанавливается за ремонт
      (0.25 = +25% целостности за уровень)
    • combine_enabled: можно ли объединять предметы для ремонта (true)
    • combine_bonus: бонус целостности за объединение (0.1)
    • material_craft.enabled: ремонт материалами (true)
    • material_craft.integrity_per_material: +10% за материал

  features.integrity.mending_xp
    • Описание: Восстановление целостности через опыт.
    • enabled: можно ли чинить через опыт (true)
    • integrity_multiplier: сколько целостности за 1 XP (0.5)

  features.integrity.combine
    • Описание: Объединение двух одинаковых предметов.
    • enabled: можно ли объединять (true)
    • loss_rate: потери при объединении (0.0 = без потерь)

  features.integrity.xp_integrity
    • Описание: Конвертация опыта в целостность (на столе зачарования).
    • enabled: можно ли (true)
    • integrity_per_xp: +0.1% целостности за 1 XP

  features.integrity.low_integrity_warning
    • Описание: Предупреждения при низкой целостности.
    • enabled: включены (true)
    • thresholds: при каких процентах предупреждать
      [75, 50, 25, 10, 5] — при 75%, 50%, 25%, 10%, 5%

  features.integrity.unbreaking.enabled
    • Описание: Учитывать ли чары Unbreaking (замедляют износ).
    • Значения: true/false

  features.integrity.piercing
    • Описание: Чары Piercing (пробитие) ускоряют износ цели.
    • enabled: true
    • extra_integrity_cost: доп затраты целостности (0.5 = +50%)

  features.integrity.blacklist / whitelist
    • Описание: Списки материалов ДЛЯ КОТОРЫХ система целостности
      НЕ РАБОТАЕТ (blacklist) или РАБОТАЕТ ТОЛЬКО (whitelist).
      Если whitelist непустой — система работает ТОЛЬКО на этих материалах.
      [] — пустой список = работает на всех материалах.
    • Пример blacklist: ["BOW", "CROSSBOW", "FISHING_ROD"]
    • Пример whitelist: ["DIAMOND_SWORD", "NETHERITE_SWORD"]

  chat_filter.enabled
    • Описание: Фильтр запрещённых слов в чате.
      По умолчанию выключен (false). Включите, если хотите
      блокировать мат и другие нежелательные слова.
    • Значения: true / false
    • По умолчанию: false

  chat_filter.words
    • Описание: Список запрещённых слов (с поддержкой * как wildcard).
      Напр "*блять*" — блокирует любые слова, содержащие "блять".
      "*хуй" — слова, заканчивающиеся на "хуй".
      "пидор*" — слова, начинающиеся на "пидор".
      Без звёздочек — точное совпадение слова.
    • Значения: список строк

  chat_filter.regex_patterns
    • Описание: Список regex-выражений для блокировки.
      Используется, когда простых слов с * недостаточно.
      Каждый pattern — это Java regex.
      Пример: ".*[А-ЯЁA-Z]{15,}.*" — капслок из 15+ букв.
    • ВНИМАНИЕ: Если ваша версия config.yml создана ДО обновления,
      regex могут содержать \p{L} — это НОРМАЛЬНО, эти pattern'ы
      встроены в код плагина и не зависят от config.yml.

  Право байпасса: mcplugin.chat.filter.bypass

    • Описание: Кастомный скорборд (правый экран). Может иметь

    • Значения: true / false
    • По умолчанию: true

    • Описание: Интервал обновления скорборда (в тиках).
      20 = 1 раз в секунду.
    • Значения: любое целое число ≥ 5
    • По умолчанию: 20

  scoreboard.boards
    • Описание: Список конфигураций скорборда.
      Каждая board имеет имя (напр "default", "pvp", "mining").
      Условия определяют, кому и когда показывать этот скорборд.

    Каждая board:
      condition:
        type:     always / world / world_blacklist / permission
                  always — показывать всем всегда
                  world — показывать ТОЛЬКО в этих мирах
                  world_blacklist — показывать ВО ВСЕХ КРОМЕ этих миров
                  permission — показывать только с правом
        worlds:   список миров (для type=world/world_blacklist)
        permission: право (для type=permission)

      title:    Заголовок скорборда (MiniMessage с плейсхолдерами)
                Максимум ~40 символов (ограничение Minecraft)

      lines:    Строки скорборда (список, макс 15 строк)
                Пустая строка = "" = пустая линия-разделитель
                Каждая строка — MiniMessage с плейсхолдерами

  Пример нескольких board:
    boards:
      default:
        condition:
          type: always
        title: "<gradient:#00AAFF:#FF55FF>MC-Plugin</gradient>"
        lines:
          - ""
          - "<gray>Online: <white>{online}</white></gray>"
          - "<gray>TPS: <white>{tps}</white></gray>"
          - ""
          - "<gray>Your ping: <white>{player_ping}ms</white></gray>"
          - "<gray>World: <white>{world_name}</white></gray>"

      pvp:
        condition:
          type: world
          worlds: ["pvp_arena"]
        title: "<red>⚔ PVP ARENA</red>"
        lines:
          - "<gray>Kills: <white>{kills}</white></gray>"
          - "<gray>Deaths: <white>{deaths}</white></gray>"

╚══════════════════════════════════════════════════════════════════╝

  chat.enabled
    • Описание: Кастомный чат. Заменяет ванильный чат на форматированный
      с поддержкой MiniMessage, плейсхолдеров и групповых форматов.
    • Значения: true / false
    • По умолчанию: true

  chat.mode
    • Описание: Режим работы чата.
    • static:     один формат для всех (из chat.format)
    • per-group:  разные форматы для разных групп (из chat.groups.formats)
                  Определяется через {group} плейсхолдер (LuckPerms)
    • per-world:  разные форматы для разных миров (из chat.worlds)
    • По умолчанию: "static"

  chat.format
    • Описание: Формат сообщения в чате (для static mode).
      MiniMessage + плейсхолдеры + {message} (текст сообщения).
      Пример: "<gray>{player_name}</gray>: <white>{message}</white>"
    • По умолчанию: "<green>ᴄʜᴀᴛ <dark_gray>» <white>{player_name}<gray>: <white>{message}"

  chat.player_minimessage
    • Описание: Разрешить ли игрокам использовать MiniMessage-теги
      в своих сообщениях (цвета, градиенты и т.д.).
      ВНИМАНИЕ: если true — игроки могут использовать любые цвета.
    • Значения: true / false
    • По умолчанию: true

  chat.message_placeholders
    • Описание: Разрешать ли плейсхолдеры в тексте сообщений игроков.
      Если true — {online} в сообщении игрока будет заменён на онлайн.
    • Значения: true / false
    • По умолчанию: true

  chat.bypass_permission
    • Описание: Право для полного байпасса кастомного чата.
      Игроки с этим правом могут писать в ванильный чат (если нужно).
      Пустая строка "" — ни у кого нет байпасса.
    • По умолчанию: ""

  chat.groups.default
    • Описание: Формат чата по умолчанию (для групп, которых нет в formats).
      Используется, если у игрока нет совпадения по группам.
    • По умолчанию: "<gray>[<white>{player_name}</white>]</gray> <white>{message}</white>"

  chat.groups.formats.<группа>
    • Описание: Форматы для конкретных групп LuckPerms.
      Ключ = название группы (из LuckPerms), значение = формат.
      Пример:
        admin: "<red>[Admin]</red> <white>{player_name}</white>: <red>{message}</red>"
        vip: "<gold>[VIP]</gold> <white>{player_name}</white>: <gold>{message}</gold>"
        default: "<gray>{player_name}</gray>: <white>{message}</white>"

  chat.worlds.<мир>
    • Описание: Форматы для разных миров (если mode = per-world).
      Ключ = название мира, значение = формат.
      Пример:
        world: "<green>{player_name}</green>: <white>{message}</white>"
        world_nether: "<red>{player_name}</red>: <white>{message}</white>"

  access_control.check_interval_ticks
    • Описание: Интервал ПЕРИОДИЧЕСКОЙ ПРОВЕРКИ всех онлайн-игроков
      на соответствие whitelist/blacklist/opwhitelist.
      20 тиков = каждую секунду.

╔══════════════════════════════════════════════════════════════════╗

║   30. 📡 БЕСПРОВОДНОЙ РЕДСТОУН (wireless_redstone)              ║
║                                                                  ║
╚══════════════════════════════════════════════════════════════════╝

  wireless_redstone.enabled
    • Описание: Беспроводная передача редстоун-сигнала между блоками.
      Позволяет связать редстоун-лампы, наблюдатели, поршни, диспенсеры
      и редстоун-пыль БЕЗ проводов.
      Один блок может быть связан с НЕСКОЛЬКИМИ другими (one-to-many).
    • Значения: true / false
    • По умолчанию: true

  Использование:
    1. Shift+ПКМ по первому устройству → "Binding" в actionbar
    2. Shift+ПКМ по второму устройству → связаны!
    3. Повторить с третьим → все три связаны (one-to-many)
    4. Shift+ПКМ по устройству с уже существующими связями →
       показывает количество связей и позволяет добавить ещё
    5. Shift+ПКМ по тому же блоку дважды → отмена привязки
    6. Сломать устройство → все его связи разрываются

  Поддерживаемые блоки:
    - REDSTONE_LAMP (лампа)
    - OBSERVER (наблюдатель)
    - PISTON / STICKY_PISTON (поршень)
    - DISPENSER / DROPPER (раздатчик)
    - REDSTONE_WIRE (редстоун-пыль)

  Каскадная активация:
    Лампа A → (беспровод) → Лампа B → (беспровод) → Лампа C.
    При включении A включаются B и C автоматически (с защитой от циклов).

  Примечание: Связи хранятся в БД (таблица wireless_links) и сохраняются
  между перезапусками сервера. Чанки с привязанными блоками держатся
  загруженными через plugin chunk tickets.

╔══════════════════════════════════════════════════════════════════╗
║                                                                  ║
║   ПЛЕЙСХОЛДЕРЫ — ПОЛНЫЙ СПРАВОЧНИК                              ║
║                                                                  ║
╚══════════════════════════════════════════════════════════════════╝

  Все плейсхолдеры разрешаются через PlaceholderResolver.
  Работают в формате MiniMessage: цвета, градиенты, теги.
  Порядок разрешения: Встроенные → Ping → Статистика → PAPI (последние).

  ────────────────────────────────────────────────────────────────
  1. 👤 ПЛЕЙСХОЛДЕРЫ ИГРОКА (без PAPI)
  ────────────────────────────────────────────────────────────────

  {player_name}           — ник игрока
  {player_displayname}    — отображаемое имя (с префиксами/суффиксами)
  {player_uuid}           — UUID игрока
  {player_ping}           — пинг в мс (только число)
  {player_ping_color}     — пинг с HEX-цветом (MiniMessage <#RRGGBB>)
  {player_ping_gradient}  — пинг с MiniMessage-градиентом
  {player_gamemode}       — режим игры (survival/creative/...)
  {player_health}         — здоровье (число, например 20.0)
  {player_food}           — сытость (число, 0-20)
  {player_level}          — уровень опыта
  {player_xp}             — прогресс опыта (0-100%)

  ────────────────────────────────────────────────────────────────
  2. 🌍 ПЛЕЙСХОЛДЕРЫ МИРА (без PAPI)
  ────────────────────────────────────────────────────────────────

  {world_name}            — название текущего мира игрока
  {world_players}         — количество игроков в текущем мире

  ────────────────────────────────────────────────────────────────
  3. 🖥 ПЛЕЙСХОЛДЕРЫ СЕРВЕРА (без PAPI)
  ────────────────────────────────────────────────────────────────

  {online}                — текущий онлайн
  {max_players}           — максимальное количество игроков
  {tps}                   — TPS (форматированное, 0.00)
  {tps_color}             — TPS с HEX-цветом (зелёный/жёлтый/красный)
  {mspt}                  — MSPT (миллисекунды на тик)
  {mspt_color}            — MSPT с HEX-цветом
  {ram}                   — RAM в % (напр "45.2")
  {ram_color}             — RAM с HEX-цветом
  {uptime}                — аптайм сервера (напр "12d 5h 30m")
  {server_name}           — имя сервера (Bukkit.getName())
  {server_version}        — версия сервера (напр "1.21.4")

  ────────────────────────────────────────────────────────────────
  4. 🔤 ВСТРОЕННЫЕ LUCKPERMS (без PAPI)
  ────────────────────────────────────────────────────────────────

  {prefix}                — префикс из LuckPerms (если установлен)
  {suffix}                — суффикс из LuckPerms
  {group}                 — название первичной группы LuckPerms

  ⚠ Для работы нужен LuckPerms на сервере. Автоопределение при старте.

  ────────────────────────────────────────────────────────────────
  5. ⏱ ДИНАМИЧЕСКИЕ ПЛЕЙСХОЛДЕРЫ (статистика за период)
  ────────────────────────────────────────────────────────────────

  Позволяют получить min/avg/max метрики за временной период.
  Статистика собирается раз в секунду (буфер: ~24 часа).

  Формат: {<метрика>[_min|max|avg]_<время>[_color]}

  Метрики:
    tps     — тики в секунду
    mspt    — миллисекунды на тик
    online  — количество игроков
    ram     — использование RAM (%)

  Формат времени:
    1s  — за последнюю 1 секунду
    30s — за последние 30 секунд
    5m  — за последние 5 минут
    1h  — за последний 1 час
    1d  — за последний 1 день
    ss  — за всю серверную сессию

  Суффиксы:
    _min    — минимальное значение за период
    _max    — максимальное значение за период
    _avg    — среднее значение (по умолчанию, можно опустить)
    _color  — значение с HEX-цветом

  Примеры:
    {tps}             — TPS сейчас
    {tps_5m}          — средний TPS за 5 минут
    {tps_min_1h}      — минимальный TPS за час
    {tps_max_ss}      — максимальный TPS за сессию
    {tps_1s_color}    — TPS за 1 сек с цветом
    {mspt_max_1h}     — максимальный MSPT за час
    {online_max_1h}   — максимальный онлайн за час
    {ram_avg_5m}      — средняя RAM за 5 минут
    {ram_color}       — RAM с цветом

  ────────────────────────────────────────────────────────────────
  6. 📶 ПИНГ-ПЛЕЙСХОЛДЕРЫ
  ────────────────────────────────────────────────────────────────

  Формат: {ping[_min|max|avg]_<время>_<scope>[_color]}

  Scope (область):
    all     — средний пинг ВСЕХ онлайн игроков
    ys      — свой пинг (текущий, без истории)
    <ник>   — пинг конкретного игрока (только текущий)

  Примеры:
    {ping}              — средний пинг всех игроков
    {ping_5m}           — средний пинг всех за 5 минут
    {ping_ys}           — свой текущий пинг
    {ping_all}          — средний пинг всех сейчас
    {ping_min_30s_all}  — минимальный пинг всех за 30 сек
    {ping_max_1h_all}   — максимальный пинг всех за час
    {ping_ys_color}     — свой пинг с градиентом
    {ping_1s_Steve}     — пинг игрока Steve
    {ping_Steve_color}  — пинг Steve с цветом

  Цвета пинга:
    0-200ms     — зелёный     (#00AA00 → #55FF55)
    200-400ms   — жёлтый      (#55FF55 → #FFFF55)
    400-600ms   — оранжевый   (#FFFF55 → #FFAA00)
    600-800ms   — красный     (#FFAA00 → #FF5555)
    800-1000ms+ — тёмно-красный (#FF5555 → #AA0000)

  ────────────────────────────────────────────────────────────────
  7. 🟢 PAPI (PlaceholderAPI) — ЛЮБЫЕ ВНЕШНИЕ ПЛЕЙСХОЛДЕРЫ
  ────────────────────────────────────────────────────────────────

  Если на сервере установлен PlaceholderAPI, любые PAPI-плейсхолдеры
  работают наравне со встроенными.

  Примеры:
    %luckperms_prefix%           — префикс из LuckPerms
    %luckperms_suffix%           — суффикс из LuckPerms
    %luckperms_primary_group_name%  — название группы
    %player_world%               — название мира
    %player_xp%                  — опыт
    %vault_eco_balance%          — баланс (Vault Economy)
    %player_health%              — здоровье (PAPI версия)

  PAPI-плейсхолдеры разрешаются ПОСЛЕ встроенных, можно комбинировать:
    {player_name} %luckperms_prefix%

  ────────────────────────────────────────────────────────────────
  8. 🔔 ПИНГИ В ЧАТЕ (@mentions)
  ────────────────────────────────────────────────────────────────

  Работают в сообщениях чата (не в формате, а в тексте сообщения).

  @everyone       — пинг всех онлайн (кроме себя)
  @<ник>          — пинг конкретного игрока (напр @Steve)
  @non-op         — пинг всех без OP
  @is-admin       — пинг всех с правом mcplugin.admin/mcplugin.*
  @is-non-admin   — пинг всех без админ-прав

  Пропингованные игроки слышат звук и получают уведомление.
  Стиль пинга настраивается в config.yml → chat_ping.ping_style

  ───── ВАНИЛЬНЫЙ /whitelist ─────
  Ванильная команда /whitelist ЗАБЛОКИРОВАНА.
  Используйте: /mp whitelist on|off|add|remove|list

  ───── ВАНИЛЬНЫЙ /op и /deop ─────
  Ванильные /op и /deop ЗАБЛОКИРОВАНЫ.
  Используйте: /mp chgop (графический интерфейс)

  ───── ВАНИЛЬНЫЙ /stop и /restart ─────
  Перехвачены плагином (с подтверждением).
  Используйте: /mp power off и /mp power reboot

  ───── БАЗА ДАННЫХ ─────
  Все данные хранятся в plugins/MC-Plugin/database.db (SQLite).
  Рекомендуется регулярное резервное копирование!

  ───── ДАТАПАК ─────
  Плагин автоматически устанавливает датапак MC-Datapack
  в папку world/datapacks/ при первом запуске.
  Датапак содержит кастомные рецепты, функции и структуры.

  ───── ПЕРЕЗАГРУЗКА ─────
  После изменения config.yml используйте /mp reload.
  Это перезагрузит конфиги и переинициализирует модули.
  Полная перезагрузка сервера НЕ ТРЕБУЕТСЯ (но возможна).
